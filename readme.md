# scCTNG Documentation

This repository is a comprehensive analytical framework based on the original **AE-CTNG** algorithm. It integrates **ZINB-Autoencoder dimensionality reduction** with an **improved CTNG clustering** method. By setting `AUTO_IMPROVE = True` in `run_new.py`, the program enters "Intelligent Diagnosis Mode," automatically selecting the optimal optimization strategy based on data characteristics.

---
## Requirements
- Python 3.8+
- numpy
- scipy
- torch
- scanpy
- scikit-learn

Install dependencies via:
pip install -r requirements.txt

## 1. Quick Start

1.  **Environment Setup**: Ensure the directory contains all dependency files: `AE.py`, `preprocess.py`, `search_TN_optimized.py`, `divide.py`, `clustering.py`, `BestMapping.py`, `metrics.py`, and `improvements.py`.
2.  **Prepare Data**: Place your datasets in the `datasets/[dataset_name]/` folder.
3.  **Configure Parameters**: Open `run_new.py` and modify the `filename` (dataset name) as needed.
4.  **Run**:
    ```bash
    python run_new.py
    ```

---

## 2. Core Workflow

Upon startup, the program first checks for existing dimensionality-reduced data (Latent Space). If not found (or if forced retraining is enabled), the execution proceeds in two phases:

* **Phase 1: ZINB Pre-training**: Trains the Autoencoder using the ZINB (Zero-Inflated Negative Binomial) loss function to obtain the initial latent space representation.
* **Intelligent Diagnosis**: Performs a rapid clustering pass in the initial space. If the **Initial Cluster Count > `FRAG_THRESHOLD`**, the data is diagnosed as "Space Fragmentation," automatically triggering deep optimization.
* **Phase 2: Joint Optimization (Optional)**: If the diagnosis is triggered, KL-divergence constraints are enabled to update both pseudo-labels and network weights simultaneously.
* **Final Phase: K-Scanning (drecord)**: Scans a range of $k$ values in the final latent space to find the optimal clustering results and calculate performance metrics.

## 3. Key Parameter Controls

You can manually adjust the following key parameters in the configuration section of `run_new.py` (after `if __name__ == "__main__":`):

### A. Scan Range (`K_RANGE`)
Controls the search interval for $k$ during the final result evaluation.
* **Code Location**: `K_RANGE = range(3, 41)`
* **Function**: The $k$ value defines the local density. Scanning this range helps identify the $k$ that best fits the true cluster count of the dataset.

### B. Diagnosis Threshold (`FRAG_THRESHOLD`)
Controls the sensitivity of the "Auto-Diagnosis."
* **Code Location**: `FRAG_THRESHOLD = 50`
* **Logic**:
    * If the pre-trained data is too scattered (producing > 50 initial micro-clusters), the program identifies the data quality as poor and forces **Improvement 1 (KL Joint Optimization)**.
    * To enable deep optimization for all datasets regardless of quality, lower this value (e.g., to `10`); to bypass it, increase the value.

### C. Improvement Strategy (`AUTO_IMPROVE = True`)
When enabled, the logic is as follows:
* **Improvements 2 & 3**: **Always Enabled**. These add minimal computational overhead while providing stable boosts to clustering accuracy.
* **Improvement 1**: **Triggered by Diagnosis**. Since KL joint optimization involves model retraining and is time-consuming, it is only activated when severe fragmentation is detected.

### D. Latent Space Reuse (`RETRAIN_AE`)
Controls whether to skip the dimensionality reduction phase and load existing data.
* **Logic**:
    * **`False`**: The program searches for an existing `..._latent_zinb.csv` file. If found, it skips Phase 1 and Phase 2, jumping straight to the K-Scanning phase.
    * **`True`**: Forces the AE to retrain regardless of existing files.

## 4. Description of Improvement Modules

### Improvement 1: KL Joint Optimization
* **Function**: Implements "clustering-oriented" dimensionality reduction.
* **Principle**: Introduces KL-divergence alongside the ZINB loss. Using the labels generated by CTNG as pseudo-labels (Target Distribution), it forces the Encoder to compress intra-cluster distances and expand inter-cluster distances.
* **Note**: Runs only when `IS_FRAGMENTED` is `True` and `AUTO_IMPROVE` is enabled.

### Improvement 2: Adaptive Beta
* **Function**: Replaces the global `BASE_BETA` used in the original algorithm.
* **Principle**: Dynamically calculates local density thresholds based on the distance distribution of the $k$-nearest neighbors for each cell. This allows the algorithm to handle "high-density" and "sparse" regions simultaneously, significantly boosting NMI.

### Improvement 3: Smart Merge
* **Function**: Solves the problem of CTNG producing excessive "fragmented clusters" in noisy regions.
* **Principle**: Calculates the centroid distance between micro-clusters (controlled by `MIN_SIZE_RATIO`) and their neighbors in the latent space, automatically merging them into the most similar neighbor.

---

## 5. Output

Results are saved in the following directories:
* **Latent Space**: `DimensionalityReduction/[dataset_name]/` (Contains `.csv` latent representations and pseudo-labels).
* **Scan Results**: `result/drecord_result/[dataset_name]/` (Contains `k_clusters.csv` tables and `k_clusters.png` trend charts).
* **Metrics**: The terminal displays real-time ACC, FM, NMI, and ARI values for each $k$, outputting the optimal result with the highest comprehensive score at the end.

---

### Dev Note: Manual Control for Ablation Studies
If you need to perform ablation studies, set `AUTO_IMPROVE = False`. You can then:
* Toggle KL Joint Optimization via `USE_IMPROVE1 = True/False`.
* Toggle Adaptive Beta via `USE_IMPROVE2 = True/False`.
* Toggle Smart Merge via `USE_IMPROVE3 = True/False`.
